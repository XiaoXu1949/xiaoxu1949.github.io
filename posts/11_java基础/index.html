接口、内部类

2021.7.7

接口

  • 接口就像抽象abstract的一种具体运用

  • interface 接口名{ //属性（public static final）(必须初始化) //方法}

  • class 类名 implements 接口 { 自己的属性；自己的方法；必须实现的接口的抽象方法；}

  • jdk8 以后接口可以有静态方法(static)，默认方法(default)

  • implements 也包含有继承，也就是用来实现的类，可以继承接口中的属性（public static final）

  • 细节：

    • 接口不能被实例化

    • 接口中所有的方法是public方法，接口中抽象方法，可以不用abstract修饰

    • 一个普通类实现接口，就必须将该接口的所有方法都实现，可以使用alt + enter 来快速实现

    • 抽象类 实现接口，可以不用实现接口的方法

    • 一个类可以同时实现多个接口

    • 接口中的属性，只能是final的，而且是public static final (省略的) 修饰【如：public static final int a = 1；】(必须初始化)

    • 接口中属性的访问形式：接口名.属性名

    • 接口不能继承其它类，但是可以继承多个别的接口

    • 接口的修饰符只能是public和默认，这点和类的修饰符一样

  • 接口实现VS继承。当子类继承了父类，就自动的拥有了父类的功能；如果子类需要扩展功能，可以通过实现接口的方式扩展；可以理解 实现接口 是对java 单继承机制的一种补充

  • 接口和继承解决的问题不一样

    • 继承的价值主要在于：解决代码的复用性和可维护性
    • 接口的主要价值在于：设计，设计好各种规范(方法)，让其它类实现这些方法。即，更加灵活

  • 接口比继承更加灵活，继承是满足 is - a 的关系，而接口只需满足 like - a 的关系

  • 接口在一定程度上实现代码解耦【即 接口的规范性 + 动态绑定机制】

  • 接口的多态特性

    • 多态参数，接口引用可以指向实现了接口的类的对象
    • 多态数组
    • 接口存在多态传递现象（接口可以继承接口）

内部类

  • 一个类的内部又完整的嵌套了另一个类结构。被嵌套的类称为内部类，嵌套其它类的类称为外部类。内部类的最大特点就是可以直接访问私有属性，并且可以体现类与类之间的包含关系

  • 内部类的四种分类

    • 定义在外部类局部位置上(比如方法内)

      1. 局部内部类(有类名)

      2. 匿名内部类（没有类名）（重点！！！！！！）

    • 定义在外部类的成员位置上 3. 成员内部类（没有static修饰）

      1. 静态内部类（使用static修饰）

局部内部类

  • 局部内部类是定义在外部类的局部位置，比如方法中，并且有类名

  1. 可以访问外部类的所有成员，包括私有的

  2. 不能添加访问修饰符，因为它的地位就是一个局部变量。局部变量是不能使用修饰符的。但是可以使用final修饰，因为局部变量也可以使用final

  3. 作用域：仅仅在定义它的方法或代码块中

  4. 局部内部类—访问—>外部类成员 【直接访问】

  5. 外部类—访问—>局部内部类成员 访问方式：创建对象，再访问(注意：必须在作用域内)

  • 记住三句话：
    1. 局部内部类定义在方法中或代码块中
    2. 作用域在方法体或代码块中
    3. 内部类的本质仍然是一个类

  1. 外部其它类—不能访问—->局部内部类(因为 局部内部类地位是一个局部变量)

  2. 如果外部类和局部内部类的成员重名时，默认遵循就近原则，如果想访问外部类的成员，则可以使用【外部类名.this.成员】去访问

匿名内部类

  • 匿名内部类使用（！！！！！！！！！！）
  • 抓手
    1. 本质是类
    2. 内部类
    3. 该类没有名字
    4. 同时还是一个对象
  • 语法： new 类或接口 (参数列表) {类体}；
  • 匿名内部类继承了你基于的类或接口，然后马上new了一个实例化，然后你基于的类或接口引用指向这个实例
  • 匿名内部类的最佳实践—> 当做实参直接传递！！！

成员内部类

  • 方法中不能再定义方法，成员内部类相当于类中的方法，因此不能在成员内部类中再定义内部类
  • 成员内部类是定义在外部类的成员位置，并且没有static修饰
  • 其它特点类似上面

静态内部类

  • 成员内部类是定义在外部类的成员位置，有static修饰
  • 其它特点类似上面

学习里的听和做

别贪不属于自己的东西

20216.20

刚一个不认识的实验室学姐回实验室收拾东西，忽然尴尬的发现自己贪了她的书。事情是这样的，研三的一个学姐读博去了，留下一个位置，以为没人坐了，桌上有几本书，三本linux的书，还有三本行草。本着肥水不流外人田，便没有经过物主人的同意，擅自拿走了。

今天物主人回来了，还在找这些书，虽然她不知道在我这里，但自己觉得惭愧，恨不得找个地缝钻进去，就是贪小便宜。事情没有败露，却给自己提个醒，见微知著。不是自己的东西不要妄想占有，不要贪小便宜，只拿属于自己的东西就好，也不要贪得无厌，多了不一定好。

回头看，拿了哪些书，又有什么用呢！都没怎么看过，不看何必要占有哪些书呢？竟然占有了，就要物尽其用。现代人就是过度消费，消费这些物质，消费感情。再豪华的房子，住着不安心；再奢侈的轿车，开着不踏实；房间里堆再多的书，不看；手里拿着再多的钱，没有真感情；这又有什么用呢！看似富足，实则贫乏。

做到光明磊落，心如止水，坦坦荡荡。这些更多的是心，你的心也即映射了这个世界。要做到我心光明，我心光明了，任何事情都不要害怕，上天会帮助你的！

2021.5.21向他汇报

这次汇报时，心情极度不好。

这次汇报，自己没有什么成果、进展，又为小论文的事情着急。老师又一个劲的催着做一些自己很难做出来得东西，因此导致心情极度沮丧，晚上给老妈通了电话，心情才有所平复。

当时写的日记：

今天又和他汇报，没啥用，感觉他也是在完成任务。为什么呢？就是没有实质性的帮助，再加上他说的话，给的命令，完全就没啥帮助。我希望有张科备那种实打实的帮助，或者郭淼那种精神上的帮助都可以。然而，我再怎么严格要求自己，他都无感。或许他自身难保，自己的事情都处理不了呢！不管了！按照自己的进度来就好了。

按照自己的进度来，事情要做，但是要按照我自己的节奏来，不管你说什么，除非有实质性的帮助，不然都按照我自己的方式来做。目标不会变了，就是这个追踪器加分割目标。数据集有了，最大的麻烦是啃下代码，代码怎么实现的，以及实现的原理。然后才是怎么修改，实验。实验是一小部分。按照实际情况，应该是先写，把该写的东西写好，在写的基础上，做实验，就是把写的东西弄好看，再给一点依据。

做吧，按照自己的方法，按照自己的节奏来！

总结：首先自己要努力，内源动力最总要。其次，对于别人的要求 量力而行，要结合自己的实际来做事，有自己的节奏最重要。最后，不管怎么样，都是要自己对自己负责滴，父母再厉害也只能帮你某些方面；父母再不厉害，也能在某些方面帮助你。言而总之，尽人事，知天命。每个人的能力都是有限的，尽力去做了，享受了过程，对于结果，不可控，秉承着顺其自然，船到桥头自然直就好。
