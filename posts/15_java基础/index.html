集合

数组VS集合

  • 数组的缺点

    1. 长度开始时必须指定，而且一旦指定，不能修改
    2. 保存的必须为同一类型的元素
    3. 使用数组进行增加/删除元素比较麻烦

  • 集合的好处

    1. 可以动态保存任意多个对象
    2. 提供了一系列方便操作对象的方法：add、remove、set、get等
    3. 使用集合添加、删除新元素代码变得简洁

集合的框架体系

  • 集合主要是两组(单列集合 , 双列集合)
  • Collection 接口有两个重要的子接口 List 、Set , 他们的实现子类都是单列集合
  • Map 接口的实现子类 是双列集合，存放的 K-V

Collection 接口

Collection 接口实现类的特点

  1. collection实现子类可以存放多个元素，每个元素可以是Object
  2. 有些Collection的实现类，可以存放重复的元素，有些不可以
  3. Collection的实现类，有些是有序的(List),有些是无序的（Set）
  4. Collecction接口没有直接的实现子类，是通过它的子接口Set和List来实现的

List 接口和常用方法

List 接口基本介绍

  • List接口是Collection接口的子接口
    1. List集合类中元素有序(即添加顺序和取出顺序一致)、且可以重复
    2. List集合中的每个元素都有其对应的顺序索引，即支持索引
    3. List容器中的元素都对应一个整数型的序号记载其在容器中的位置，可以根据序号存取容器中的对应元素
    4. JDK API中List接口的实现类有许多，常用的有ArrayList、LinkedList、Vector

1.ArrayList 的注意事项

  1. ArrayList可以加入null，并且多个
  2. ArrayList是由数组来实现数据存储的
  3. ArrayList基本等同于Vector，除了ArrayList是线程不安全(执行效率高)，在多线程情况下，不建议使用ArrayList

ArrayList 的底层操作机制源码分析(重点，难点.)

  1. ArrayList中维护了一个Object类型的数据（数组）elementData transient Object[] elementData

    • transient 表示瞬间的，短暂的，表示该属性不会被序列化

  2. 当创建ArrayList对象时，如果使用的是无参构造器，则初始elementData容量为0，第一次添加，则扩容elementData为10，如需要再次扩容，则扩容elementData为1.5倍

  3. 如果使用的是指定大小的构造器，则初始elementData容量为指定大小，如果需要扩容，则直接扩容elementData为1.5倍

2.Vector 底层结构和源码剖析

  • Vector底层也是一个对象数组，protected Object[] elementData
  • Vector是线程同步的，即线程安全，Vector类的操作方法带有synchronized
  • 在开发中，需要线程同步安全时，考虑使用Vector

Set 接口和常用方法

Set 接口基本介绍

  • 无序（添加和取出的顺序不一致），没有索引

  • 不允许重复元素，所以最多包含一个null

  • 和 List 接口一样, Set 接口也是 Collection 的子接口，因此，常用方法和 Collection 接口一样.

Set 接口的遍历方式

  1. 可以使用迭代器
  2. 增强for
  3. 不能使用索引的方式来获取

Set 接口的常用方法

Set set = new HashSet();
set.add("john");
System.out.println("=====使用迭代器====");
Iterator iterator = set.iterator();
while (iterator.hasNext()) {
     Object obj = iterator.next();
     System.out.println("obj=" + obj);
}
set.remove(null);

Set 接口实现类-HashSet

  1. HashSet实现了Set接口
  2. HashSet实际上是HashMap
  3. 可以存放null值，但是只能有一个null
  4. HashSet不保证元素是有序的，取决于hash后，再确定索引的结果（即，不保证存放元素的顺序和取出顺序一致）
  5. 不能有重复元素/对象。
  6. 底层维护了一个数组+（多个）单向链表

HashSet 底层机制说明

  • hash值不同时，也可能在table表的同一个位置，由于哈希值不同，则不会判断equals()，会直接加在该位置后面，形成单向链表

Set 接口实现类-LinkedHashSet

  • 底层是LinkedHashMap，维护了一个数组+（一个）双向链表
  • LinkedHashSet是HashSet的子类
  • LinkedHashSet根据元素的hashCode值来决定元素的存储位置，同时使用链表维护元素的次序，这使得元素看起来是以插入的顺序保存的
  • LinkedHashSet不允许添加重复元素

刚一个不认识的实验室学姐回实验室收拾东西，忽然尴尬的发现自己贪了她的书。事情是这样的，研三的一个学姐读博去了，留下一个位置，以为没人坐了，桌上有几本书，三本linux的书，还有三本行草。本着肥水不流外人田，便没有经过物主人的同意，擅自拿走了。

今天物主人回来了，还在找这些书，虽然她不知道在我这里，但自己觉得惭愧，恨不得找个地缝钻进去，就是贪小便宜。事情没有败露，却给自己提个醒，见微知著。不是自己的东西不要妄想占有，不要贪小便宜，只拿属于自己的东西就好，也不要贪得无厌，多了不一定好。

回头看，拿了哪些书，又有什么用呢！都没怎么看过，不看何必要占有哪些书呢？竟然占有了，就要物尽其用。现代人就是过度消费，消费这些物质，消费感情。再豪华的房子，住着不安心；再奢侈的轿车，开着不踏实；房间里堆再多的书，不看；手里拿着再多的钱，没有真感情；这又有什么用呢！看似富足，实则贫乏。

做到光明磊落，心如止水，坦坦荡荡。这些更多的是心，你的心也即映射了这个世界。要做到我心光明，我心光明了，任何事情都不要害怕，上天会帮助你的！

2021.5.21向他汇报

这次汇报时，心情极度不好。

这次汇报，自己没有什么成果、进展，又为小论文的事情着急。老师又一个劲的催着做一些自己很难做出来得东西，因此导致心情极度沮丧，晚上给老妈通了电话，心情才有所平复。

当时写的日记：

今天又和他汇报，没啥用，感觉他也是在完成任务。为什么呢？就是没有实质性的帮助，再加上他说的话，给的命令，完全就没啥帮助。我希望有张科备那种实打实的帮助，或者郭淼那种精神上的帮助都可以。然而，我再怎么严格要求自己，他都无感。或许他自身难保，自己的事情都处理不了呢！不管了！按照自己的进度来就好了。

按照自己的进度来，事情要做，但是要按照我自己的节奏来，不管你说什么，除非有实质性的帮助，不然都按照我自己的方式来做。目标不会变了，就是这个追踪器加分割目标。数据集有了，最大的麻烦是啃下代码，代码怎么实现的，以及实现的原理。然后才是怎么修改，实验。实验是一小部分。按照实际情况，应该是先写，把该写的东西写好，在写的基础上，做实验，就是把写的东西弄好看，再给一点依据。

做吧，按照自己的方法，按照自己的节奏来！

总结：首先自己要努力，内源动力最总要。其次，对于别人的要求 量力而行，要结合自己的实际来做事，有自己的节奏最重要。最后，不管怎么样，都是要自己对自己负责滴，父母再厉害也只能帮你某些方面；父母再不厉害，也能在某些方面帮助你。言而总之，尽人事，知天命。每个人的能力都是有限的，尽力去做了，享受了过程，对于结果，不可控，秉承着顺其自然，船到桥头自然直就好。

遇到的问题有：

1.blog根目录下的几个文件夹是什么？是由hugo生成的。

2.git、hugo这些我安装的东西都算啥？是平常用的运行程序，还是啥?。应该都是运行程序吧，因为都用的是.exe程序安装的，并且都配置了环境变量。只不过它们都没有图形界面，但是都可以在机器上使用它们的命令。这里的机器就是指人机交互窗口(命令提示符/cmd/power shell),这些终端是介于底层硬件和操作系统之间的。

3.再就是主题，主题的那些文件夹是啥？如何看主题源码，更改配置，再进阶一点就是自己来做优化。
